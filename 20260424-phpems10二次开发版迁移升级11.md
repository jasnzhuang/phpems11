## 三、部署时的关键点

### 3.1 数据库变更（必须执行）

#### ① 添加 `questiontag` 字段

如果使用的是已有的 phpems11 数据库（而非全新导入 `phpems11.sql`），需要手动执行：

```sql
ALTER TABLE `x2_questions` 
ADD COLUMN `questiontag` varchar(72) CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci DEFAULT NULL 
AFTER `questionlevel`;
```

如果全新导入 `phpems11.sql`，则该字段已包含在 SQL 文件中。

#### ② 注册数据分析模块

访问以下 URL 完成模块注册（向 `x2_app` 表插入记录）：

```
https://您的域名/register_dataanalysis.php
```

注册成功后，后台导航中才会显示"数据分析"入口。

---

### 3.2 配置文件（必须设置）

#### DeepSeek API 密钥

编辑 `deepseek_config.php`，填入有效的 DeepSeek API 密钥：

```php
$deepseek_api_key = 'sk-您的实际密钥';
```

或通过环境变量注入（推荐）：

```bash
export DEEPSEEK_API_KEY='sk-您的实际密钥'
```

**生产环境注意**：
- 确保 `deepseek_config.php` 文件权限设置为不可外部访问（建议 `644` 或更低）
- 不要将该文件提交到公共代码仓库

---

| `data/cache/` | 系统缓存 |

## 四、常见问题与排查

### Q1: AI 出题点击后提示"你没有权限和预算使用此功能"

**原因**：权限白名单控制。  
**解决**：在 `questions.master.php` 的 `isAIQuestionFeatureAllowed()` 方法中，将当前管理员用户名加入 `$allowedUsers` 数组。

### Q2: AI 出题时提示"AI服务暂时不可用"

**原因**：DeepSeek API 密钥未配置或无效。  
**解决**：检查 `deepseek_config.php` 中的 `$deepseek_api_key`，确保密钥有效且未过期。

### Q3: 数据分析页面显示空白或报错

**原因**：`dataanalysis` 模块未注册到 `x2_app` 表。  
**解决**：访问 `register_dataanalysis.php` 完成注册。

### Q4: 成绩分析页面提示"参数错误，缺少考场ID"

**原因**：直接访问了分析 URL 而没有传 `basicid` 参数。  
**解决**：必须从"考场管理"列表点击"成绩分析"按钮进入。

### Q5: 保存 AI 生成的试题时报"Unknown column 'questiontag'"

**原因**：数据库中 `x2_questions` 表缺少 `questiontag` 字段。  
**解决**：执行 ALTER TABLE 语句添加该字段（见 3.1 节）。
